What is NetLogo Web?

NetLogo Web is a version of the NetLogo modeling environment that runs entirely in the browser. One of the primary goals of the project is to provide a suitable replacement for the woe-stricken NetLogo applet functionality, and, in doing so, once again make NetLogo models easy for NetLogo users to distribute to the world.

How do I know which edition of NetLogo is right for me?

The NetLogo and NetLogo Web are definitely very related, but, admittedly, the desktop edition of NetLogo is the one that's more polished and fully-featured. For the time being, we recommend using NetLogo Web only if one or more of the following describes your desired use of NetLogo:

  • I want to use NetLogo on a platform that it does not otherwise run on (e.g. iPad, iPhone, Android, Chromebook)
  • I want to embed a NetLogo model in a web page
  • I want to distribute a NetLogo model without requiring my audience to have an existing NetLogo installation

That is to say that NetLogo Web's primary advantage right now is how easy it makes sharing models.

Admittedly, the quality gap between NetLogo Web and NetLogo is pretty large right now, but we intend to greatly narrow that gap over the course of the coming years. Even if NetLogo Web doesn't sound helpful to you today, be sure to stay tuned as we work to improve NetLogo Web and expand its feature set.

What features are currently available in NetLogo Web?

  • Most of the NetLogo programming language
  • Plotting
  • The Command Center
  • Editable "Code" and "Info" tabs
  • Functioning widgets of all kinds and most configurations
  • The ability to export models for use in the desktop edition of NetLogo
  • The ability to export models as standalone HTML pages

What features of NetLogo does NetLogo Web lack at the moment?

  • Many of the less-common NetLogo language primitives
  • High performance
  • An interface editor
  • Extensions
  • 3D
  • HubNet
  • BehaviorSpace
  • Other peripheral NetLogo features (e.g. System Dynamics Modeler)
  • Some less-common widget configurations (e.g. vertical sliders)

That said, there are very few things in NetLogo Web that are intentionally not supported. If support for your favorite primitive or feature is not currently available, it is likely forthcoming. We appreciate your patience as we work to make NetLogo Web the best product we can.

How do I know if my model will work in NetLogo Web?

Here are a few rules for telling whether or not your model will work in NetLogo Web at the moment:

  • If your model only does the kind of standard stuff that you see in most of the sample models in the NetLogo Models Library, it is very likely that your model will work
  • If your model uses file primitives or movie primitives or extensions, it will not work
  • If your model uses wait or display, it probably won't work

If you're still not sure if your model will work or not, we encourage you to simply open your model in NetLogo Web and see for yourself. If your code uses a NetLogo primitive that isn't yet implemented, you should be given an error message that tells you so. If something else in your model isn't working correctly, NetLogo Web should make that apparent to you very quickly.

How can I create new models with NetLogo Web?

Long story short: You can't (yet).

NetLogo Web allows you to edit model code and "Info" content, but widget editing is not currently an option, which severely limits the authoring power available within NetLogo Web. Admittedly, this is one of the foremost things that we intend to improve in the near future. For now, though, all serious model-authoring tasks can only be handled in the standard, desktop edition of NetLogo.

I'm using NetLogo Web and everything's broken! What should I do?

We recommend first trying to solve the problem by refreshing the page. If that doesn't solve your problem, we encourage you to report your problem to us on GitHub or by e-mail.

I don't like the way that a particular something was done in NetLogo Web. I want to change it!

One of the great things about running directly in the browser is how easy it is to modify content. If you want to tinker with things in NetLogo Web, go for it! You're entirely free to save a standalone NetLogo Web page and create your own stylesheets or mess with the page's JavaScript—at your own risk, of course. If you think the changes you've made would be useful for others, you might even consider contributing them to NetLogo Web proper.

How can I contribute changes to the NetLogo Web project?

The relevant repositories for the project of Galapagos (for the web interface), Tortoise (for the simulation engine and compiler back-end), and NetLogo-Headless (for the compiler front-end).

Pull requests to these repositories are welcome. If you would like to talk to any of the NetLogo team about some proposed changes of yours, feel free to drop by the relevant repository's chat room on Gitter, where we'll be glad to discuss things with you.
